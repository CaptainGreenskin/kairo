# Runbook: v0.7.1 ToolResultBudget Canary and Rollback

## 目标

- 验证 ToolResultBudget 在真实流量下不会引入行为回归。
- 观察裁剪率、错误分类覆盖率、上下文压力变化。
- 满足门槛后，作为进入 `v0.8` 实施的输入结论。

## Canary Checklist

1. 灰度发布仅开启部分实例。
2. 确认 TOOL 消息出现预算元数据字段。
3. 采样检查被裁剪结果是否带尾注说明。
4. 统计 `tool_execution_failed` 等错误码分布。
5. 对比灰度前后失败率和响应时延。

## 观测指标

- `tool_result_budget_truncated_count`（每轮）
- 预算裁剪率（被裁剪工具结果数 / 工具结果总数）
- `error_code` 分布
- 模型调用失败率
- 平均迭代轮数

## 回滚条件

满足任一条件立即回滚：

- P0/P1 事故；
- 失败率显著高于基线；
- 工具结果裁剪导致关键流程不可恢复；
- 出现未知错误分类并持续放大。

## 回滚动作

1. 切回上一稳定版本或关闭新预算逻辑开关（若配置化发布）。
2. 保留灰度实例日志与样本会话。
3. 输出故障归因（预算阈值、工具异常、上下文冲突）。
4. 更新 ADR/修复清单后重新进入灰度。
