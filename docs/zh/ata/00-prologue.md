# 序：写给构建 Agent 的人

如果你正在构建一个 AI Agent——或者正在认真考虑——这个系列是为你写的。

我花了几个月时间，从零开始构建一个 Agent 运行时基础设施。过程中撞了无数面墙，做了一些事后看来还算对的选择，也留下了一堆至今没解决的问题。这个系列就是把这些东西摊开来聊。

不是产品介绍，不是技术推销。就是一个工程师的设计笔记。

起点是一道算术题。一个 20 步的 Agent，每步 90% 成功率，端到端只有 3.9%。这个数字逼着我承认一件事：Agent 挂掉，多数时候不是因为模型不够聪明，而是因为围绕模型的那层基础设施不够结实。

然后我发现，我遇到的每一个问题——资源有限怎么管、输入不可信怎么防、多个任务怎么协调、出了故障怎么恢复——五十年前的操作系统设计者全都解决过。不是巧合。是同构。

十篇文章，一条线索：

> **为什么** → **上下文** → **安全** → **工具** → **架构** → **治理** → **长任务** → **产品** → **协作** → **前沿**

不必从头读到尾。每篇都能独立看懂。但按顺序读完，你会拼出一张完整的图——关于 Agent 从 demo 走向生产，中间到底隔着什么。

我没有藏着掖着。每篇都写了还没解决的问题，有些甚至还没想清楚怎么解决。在一个每三个月就换一轮范式的领域，假装自己什么都想明白了，才是最大的问题。

开始吧。
