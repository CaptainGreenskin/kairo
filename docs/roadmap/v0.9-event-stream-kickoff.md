# Kairo v0.9 P0#2 — Event Stream Kickoff

> Companion to `docs/roadmap/v0.9-remaining-p0-scoping.md` (§3 "P0#2 Event Stream").
> Locks the design for the SSE + WebSocket Event Stream surface landing in v0.9.0 GA.
> D2 decision: **SSE + WebSocket both GA**.

## 1) Why

`KairoEventBus` is in-process. Every external consumer (ops dashboard, frontend trace view, SIEM tee, per-tenant status page) today must link Kairo into the JVM to read events. The Event Stream gate projects the bus over HTTP transports so JVM-less clients can subscribe by domain / eventType / correlation id.

Picking **two transports** (SSE + WS) at GA — instead of SSE-only with WS-later — forces the SPI to be genuinely transport-agnostic before v1.0 freezes it. If the SPI can only express SSE semantics, WS will leak into the `kairo-api` surface when added post-1.0.

## 2) Inputs / Preconditions (already in place)

- `KairoEventBus` published at `io.kairo.api.event.KairoEventBus` with `publish / subscribe / subscribe(String domain)` (v0.10 Core Refactor Wave).
- `KairoEvent` envelope with `domain`, `eventType`, `payload`, `attributes` (v0.10).
- Reactor 3.7.3 available parent-managed; Spring WebFlux 3.3.0 via `spring-boot-dependencies` BOM.
- `reactor-netty` transitive from spring-webflux — sufficient for WS handler.

## 3) Module Layout

| Module | Purpose | Spring/WebFlux dep? |
|---|---|---|
| `kairo-api` (existing) | SPI surface under `io.kairo.api.event.stream.*` | No |
| `kairo-event-stream` (new) | Transport-agnostic core: registry, back-pressure guard, filter composition | **No** — depends only on `kairo-api` + `reactor-core` |
| `kairo-event-stream-sse` (new) | SSE transport: `RouterFunction` rendering `text/event-stream` | Yes |
| `kairo-event-stream-ws` (new) | WebSocket transport: `WebSocketHandler` + JSON control/data frames | Yes |
| `kairo-spring-boot-starter-event-stream` (new) | Auto-config for both transports, independent toggles | Yes |

Four new modules + one starter. The core/transport split is the invariant that makes adding gRPC/other transports cheap post-v1.0.

## 4) SPI Surface (`kairo-api` additions)

```java
// io.kairo.api.event.stream

public interface EventStreamFilter {
    boolean test(KairoEvent event);

    default EventStreamFilter and(EventStreamFilter other) { ... }
    default EventStreamFilter or(EventStreamFilter other) { ... }

    static EventStreamFilter acceptAll() { ... }
    static EventStreamFilter byDomain(String... domains) { ... }
    static EventStreamFilter byEventType(String... eventTypes) { ... }
    static EventStreamFilter byAttribute(String key, Object expectedValue) { ... }
}

public record EventStreamSubscriptionRequest(
        EventStreamFilter filter,
        BackpressurePolicy backpressurePolicy,
        int bufferCapacity,
        Map<String, String> authorizationContext) { ... }

public enum BackpressurePolicy {
    BUFFER_DROP_OLDEST,
    BUFFER_DROP_NEWEST,
    ERROR_ON_OVERFLOW
}

public interface EventStreamSubscription {
    String id();
    Flux<KairoEvent> events();
    void cancel();
    boolean isActive();
}

// Deny-safe: must be explicitly provided by the application.
public interface KairoEventStreamAuthorizer {
    AuthorizationDecision authorize(EventStreamSubscriptionRequest request);

    record AuthorizationDecision(boolean allowed, String reason) {
        public static AuthorizationDecision allow() { return new AuthorizationDecision(true, null); }
        public static AuthorizationDecision deny(String reason) { return new AuthorizationDecision(false, reason); }
    }
}
```

All types annotated `@Experimental("EventStream — contract may change in v0.10")`.

## 5) Core Module (`kairo-event-stream`)

- `EventStreamRegistry` — tracks active `EventStreamSubscription`s; configurable global cap (default 128); rejects new subs with `TOO_MANY_SUBSCRIBERS`. Thread-safe.
- `DefaultEventStreamService` — subscribes to `KairoEventBus`, filters per-subscription, applies back-pressure policy, emits telemetry (`stream_backpressure_dropped` counter).
- `FluxBackpressureGuard` — pure Flux operator factory; policy-driven.
- Zero Spring dependency. Depends on `kairo-api` + `reactor-core` + `slf4j-api` only.

## 6) SSE Transport (`kairo-event-stream-sse`)

- `EventStreamSseEndpoint` — `@Bean RouterFunction<ServerResponse>` binding `GET /kairo/events/stream`:
  - Query params: `domain`, `eventType`, `attr.<key>=<value>`, `policy` (BUFFER_DROP_OLDEST|…), `buffer` (int).
  - Headers: `Authorization` → threaded into `authorizationContext`.
  - Calls `KairoEventStreamAuthorizer`; denies → 403 with reason in body.
  - Allowed → reactive stream of `ServerSentEvent<KairoEvent>` rendered via Jackson; heartbeat comment line every 15s to keep proxies warm.
- JSON wire format: `{"eventId","timestamp","domain","eventType","attributes",...}` — payload field omitted from wire (in-process-only).

## 7) WS Transport (`kairo-event-stream-ws`)

- `EventStreamWebSocketHandler implements WebSocketHandler` on `/kairo/events/ws`.
- Control protocol:
  - Inbound frame 1 (required): `{"type":"subscribe","domains":[...],"eventTypes":[...],"attributes":{...},"policy":"BUFFER_DROP_OLDEST","buffer":256}`
  - Subsequent outbound frames: `{"type":"event","eventId":...,"timestamp":...,"domain":...,"eventType":...,"attributes":...}`
  - Server-initiated close: control frame `{"type":"error","reason":"..."}` then close code 4403/4429/4500.
- Authorizer invoked after the subscribe frame arrives. Deny → close 4403 + reason frame.
- On disconnect, `EventStreamRegistry#cancel(subscriptionId)` cleans up.

## 8) Starter (`kairo-spring-boot-starter-event-stream`)

```properties
kairo.event-stream.enabled=false                 # master switch (default off)
kairo.event-stream.sse.enabled=false             # per-transport toggle
kairo.event-stream.ws.enabled=false              # per-transport toggle
kairo.event-stream.default-buffer-capacity=256
kairo.event-stream.default-policy=BUFFER_DROP_OLDEST
kairo.event-stream.max-subscribers=128
kairo.event-stream.sse.path=/kairo/events/stream
kairo.event-stream.ws.path=/kairo/events/ws
```

- `@ConditionalOnProperty(prefix="kairo.event-stream", name="enabled", havingValue="true")` gates the whole auto-config.
- `@ConditionalOnBean(KairoEventStreamAuthorizer.class)` on each transport — if no authorizer bean exists and a transport is enabled, **startup fails** with a clear `BeanCreationException` referencing ADR-018 deny-safe clause.
- `@ConditionalOnMissingBean` on all service/registry/endpoint beans so apps can override.

## 9) Tests (per Exit Criteria)

- `kairo-event-stream` unit:
  - Registry cap enforcement.
  - Policy behaviour per `BackpressurePolicy` value (drop-oldest, drop-newest, error).
  - `EventStreamFilter` composition (`and`, `or`, attribute match).
- `kairo-event-stream-sse` IT (`SseEventStreamIT`):
  - Spring Boot test context wired with a permissive authorizer.
  - `WebClient` subscribes; publish → receives filtered event stream; heartbeat received.
  - 403 on authorizer deny.
- `kairo-event-stream-ws` IT (`WsEventStreamIT`):
  - `WebSocketClient` connects, sends subscribe frame, receives events.
  - Disconnect → registry size drops to 0.
  - Close code 4403 on authorizer deny.
- Starter auto-config tests (`EventStreamAutoConfigurationTest`):
  - Default-off yields no beans.
  - `enabled=true` + `sse.enabled=true` + authorizer → SSE bean wired, WS absent.
  - `enabled=true` + `ws.enabled=true` + authorizer → WS bean wired, SSE absent.
  - `enabled=true` + transport on + **no** authorizer → startup fails (deny-safe).
  - User-provided beans win (`@ConditionalOnMissingBean` proofs).

## 10) Non-Goals (v0.9 cycle)

- No gRPC transport (post-v1.0 extension, SPI already transport-agnostic).
- No persistent replay / event history. Late subscribers miss past events.
- No cross-process fanout (in-process only). Clustered deployments need a bus bridge module — separate concern.
- No automatic domain-level scope enforcement beyond what the authorizer implements — authorizer is the single choke point.

## 11) ADR

- `docs/adr/ADR-018-event-stream.md` captures: transport-agnostic SPI rationale, deny-safe authorizer default, two-transport GA decision (D2), SSE-vs-WS selection guidance for consumers, back-pressure policy trade-offs.

## 12) Exit Criteria (mirrors §3 of the scoping doc)

- All tests above pass.
- `kairo-event-stream` core has zero Spring / servlet / netty dependencies.
- `mvn clean verify` green.
