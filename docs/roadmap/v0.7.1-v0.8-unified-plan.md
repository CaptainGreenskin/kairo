# Unified Plan: v0.7.1 -> v0.8

## 目标

- 先在 `v0.7.1` 修复工具结果预算问题，避免工具输出淹没上下文窗口。
- 冻结 `v0.8` 可复用契约，确保 `Execution Model` 与 `DurableExecution` 不需要重写 `v0.7.1` 产物。
- 把本次补丁的行为、错误分类、观测字段纳入标准化约定。

## 范围

### v0.7.1（本次落地）

- 在工具结果写回会话历史前执行预算治理（`ToolResultBudget`）。
- 为工具执行失败、hook 拒绝/取消补充结构化错误码。
- 在 TOOL 消息写入预算元数据，供日志与后续 replay 使用。
- 覆盖正常、超预算、异常路径测试。

### v0.8（本次冻结契约，不实现完整功能）

- 冻结执行事件模型（iteration/tool_call/tool_result/guardrail/compaction/checkpoint）。
- 冻结可恢复执行最小状态模型（cursor、history hash、pending tool calls、budget ledger）。
- 冻结 `v0.7.1` 到 `v0.8` 的兼容映射规则。

## 兼容契约（冻结）

### Tool Result Budget Contract

- TOOL 消息必须包含：
  - `tool_result_budget_applied`
  - `tool_result_budget_truncated_count`
  - `tool_result_budget_original_tokens`
  - `tool_result_budget_kept_tokens`
  - `tool_result_budget_remaining_tokens`
  - `tool_result_budget_per_result_tokens`
- 每个 `ToolResult.metadata` 必须包含：
  - `tool_result_budget_applied`
  - `tool_result_original_tokens`
  - `tool_result_kept_tokens`
  - `tool_result_truncated`
  - `tool_result_budget_reason`

### Error Classification Contract

- 工具执行失败：`error_code=tool_execution_failed`
- hook 阻断：`error_code=tool_blocked_by_hook`
- hook 取消：`error_code=tool_cancelled_by_hook`
- hook 跳过：`result_code=tool_skipped_by_hook`

### DurableExecution Compatibility

- replay 与恢复必须能读取上述预算字段与错误分类字段。
- 未识别字段按“忽略但保留”策略处理，保证前向兼容。

## 非目标

- 本阶段不引入完整持久执行状态机。
- 本阶段不引入新的 dashboard UI。
- 本阶段不改动模型调度策略。

## 验收标准

- 工具大结果在进入历史前被稳定裁剪，且保留诊断信息。
- 正常工具结果不受影响。
- TOOL 消息与 ToolResult 元数据字段齐全。
- 单测覆盖超预算和非超预算场景，并通过。
