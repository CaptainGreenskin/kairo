# Kairo v0.10 Core Refactor Waves — Verification Evidence

This document records the **release gate** outcome for the v0.10 “Core Refactor Waves” program described in the internal plan (not committed here).

## Build / Test Gate

| Command | Result | Notes |
|---|---|---|
| `mvn clean verify` | **PASS** | Full reactor build including examples + starters |
| Surefire XML aggregation | **2461** tests | Sum of `tests=` attribute across `**/target/surefire-reports/TEST-*.xml` |

## What Landed (high-signal)

### Wave 1 — Event bus + capability-shaped AgentConfig

- `KairoEventBus` + `KairoEvent` envelope (`kairo-api`)
- `DefaultKairoEventBus` (`kairo-core`)
- Optional bridging hooks:
  - `ExecutionEventEmitter` can publish to the bus (optional ctor overload)
  - `EvolutionPipelineOrchestrator` can publish lifecycle events (optional ctor overload)
  - `BusBridgingSecurityEventSink` composes logging sink + bus publish
- Capability records + builder entry points:
  - `McpCapabilityConfig`, `LoopDetectionConfig`, `DurableCapabilityConfig`
  - `AgentConfig#mcpCapability()` / `AgentConfig#loopDetection()` derived accessors
  - `AgentBuilder#loopDetection(LoopDetectionConfig)` + `AgentBuilder#durableCapability(...)`

### Wave 2 — Hook consolidation (incremental)

- `HookPhase`, `HookHandler`, `HookEvent` marker
- Existing hook event types implement `HookEvent`
- `DefaultHookChain` discovers **both** legacy annotations and `@HookHandler(phase=...)`
- Legacy Task / TeamScheduler APIs marked `@Deprecated(forRemoval=true)` with a regression test

### Wave 3 — SPI scaffolding + starter deprecation messaging

- `SkillStore` reactive SPI (minimal surface)
- `ProviderPipeline` SPI (four-stage decomposition)
- `kairo-spring-boot-starter` POM description updated to explicitly deprecate the aggregator pattern
- `MsgBuilder` in `kairo-core` marked deprecated (canonical builder remains `Msg.builder()` in API)

## Known Follow-ups (explicit non-completion)

The following items remain **intentionally open** and are tracked for a subsequent PR series:

1. **Wire `KairoEventBus` as a first-class Spring bean** and connect `BusBridgingSecurityEventSink` / optional emitters through auto-configuration (today the SPI exists; wiring is caller-driven).
2. **Delete** deprecated Task / TeamScheduler APIs after Expert Team coordinator lands.
3. **Remove MCP fields from `AgentConfig` records** after starter-only wiring is complete (today: capability accessors + deprecated builder methods).
4. **Migrate Anthropic/OpenAI providers** to implement `ProviderPipeline` end-to-end.
5. **Extract `kairo-skill` module** and physically relocate skill loader/registry/tooling.
