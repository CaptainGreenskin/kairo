# Kairo v0.9.0 — Release Verification Evidence

This document records the **release gate** outcome for the v0.9.0 GA program. It closes the
five "Locked Decisions" captured in `docs/roadmap/v0.9-remaining-p0-scoping.md` (§6) and supersedes
the earlier `docs/roadmap/v0.9-gap-only-verification.md` — which covered only the gap-closure P0s
completed before the remaining-P0 scoping was agreed.

## What Landed (high-signal)

### P0 #1 — Channel SPI (ADR-021)

- `kairo-api` channel surface: `Channel`, `ChannelInboundHandler`, `ChannelOutboundSender`,
  `ChannelMessage`, `ChannelIdentity`, `ChannelAck`, `ChannelFailureMode`.
- `kairo-channel`: `LoopbackChannel`, `ChannelRegistry`, `io.kairo.channel.tck.ChannelTCK`
  (3 scenarios — inbound dispatch / outbound `SEND_FAILED` / concurrent inbound ordering per destination).
- `kairo-spring-boot-starter-channel`: `ChannelAutoConfiguration` + `ChannelLifecycleManager`,
  deny-safe (no `ChannelInboundHandler` bean → registry only, no channels started).

### P0 #2 — Event Stream (ADR-018)

- `kairo-api/event/stream/`: `EventStreamFilter`, `EventStreamSubscription`,
  `EventStreamSubscriptionRequest`, `BackpressurePolicy`, `KairoEventStreamAuthorizer`.
- `kairo-event-stream`: `EventStreamRegistry`, `DefaultEventStreamService`,
  `FluxBackpressureGuard`.
- `kairo-event-stream-sse`: `KairoEventStreamSseHandler` (text/event-stream, filter query params).
- `kairo-event-stream-ws`: `KairoEventStreamWebSocketHandler` (JSON control frame → filter).
- `kairo-spring-boot-starter-event-stream`: single starter, independent SSE/WS toggles;
  startup fails fast without `KairoEventStreamAuthorizer` bean (deny-safe).

### P0 #3 — OTel Exporter (ADR-022)

- `kairo-observability/event/KairoEventOTelExporter`: `LoggerProvider`-based bridge,
  envelope attributes flattened under `kairo.<domain>.*`, security always-on,
  execution/team/evolution sampled via `samplingRatio`, optional key-pattern redaction.
- `kairo-spring-boot-starter-observability`: `ObservabilityAutoConfiguration`
  + `ObservabilityProperties(.EventOTel)`, deny-safe (`LoggerProvider` bean required).
- `docs/en/guide/security-observability-schema.md`: attribute schema extended
  to cover `kairo.security.*`, `kairo.execution.*`, `kairo.evolution.*`, `kairo.team.*`.

### D5 — Deprecated API deletion

- `io.kairo.api.task.*` and `TeamScheduler` physically removed from the reactor.
- All prior consumers migrated to the Expert Team coordinator (`kairo-expert-team`) and
  the hook chain introduced in v0.10. No deprecation shims remain.

## Non-Goals Preserved

- No concrete IM adapter (DingTalk / 飞书 / Slack / webhook) inside the v0.9 reactor —
  they ship as separate modules in the v0.9.x continuous-delivery train (D1).
- No historical replay / durable event storage in v0.9 — Event Stream is live-only (D2 scope-out).
- No OTel metrics / spans for bus events in v0.9 — logs only; metrics and spans come
  with ADR-023/024 once post-GA telemetry justifies their shape (D3 scope-out).
- No OTLP exporter bundled inside `kairo-observability` — applications bring their own
  concrete OTel exporter via a `LoggerProvider` bean.
