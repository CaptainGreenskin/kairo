# v0.10.2 — Structural Debt Cleanup — Verification

**State:** Released
**Scope frozen at:** 2026-04-24
**Batch:** v0.9.0 GA → v0.10.2 → v0.9.1 (DingTalk) → v1.0.0 RC1

## Goal

Land the four structural follow-ups identified in `ROADMAP.md:869-875` as hard
prerequisites for the v1.0 `@Stable` SPI freeze. Incubation-stage mandate: no
compatibility shims, no deprecation cycles.

## Scope & Evidence (four P0 items)

### B1.1 — `KairoEventBus` Spring default wiring

- `CoreAutoConfiguration` registers `KairoEventBus` as a primary `@Bean` (was
  previously unwired; only the in-process `DefaultKairoEventBus` class existed).
- Default `SecurityEventSink` wrapped with `BusBridgingSecurityEventSink` so
  every security event reaches the bus alongside the log-only sink.
- `DurableExecutionStore` implementations inject the bus and publish lifecycle
  events on `KairoEvent.DOMAIN_EXECUTION` (null-safe when the bus is absent).
- Evidence: `kairo-spring-boot-starter-core/src/main/java/io/kairo/spring/CoreAutoConfiguration.java`,
  new auto-config tests in `KairoEventBusAutoConfigurationTest` (5 tests pass).

### B1.2 — `AgentConfig` MCP field deletion

- Physically removed 4 deprecated fields from the `AgentConfig` record header:
  `mcpServerConfigs`, `mcpMaxToolsPerServer`, `mcpStrictSchemaAlignment`,
  `mcpToolSearchQuery`.
- Removed the 4 deprecated builder methods that referenced them.
- All call sites across the reactor (kairo-mcp, kairo-spring-boot-starter-mcp,
  kairo-multi-agent test rigs, `AgentContractTest`) migrated to
  `config.mcpCapability().serverConfigs()` / etc.
- Evidence: `kairo-api/src/main/java/io/kairo/api/agent/AgentConfig.java`;
  contract test `AgentContractTest.agentConfigRetainsMcpDefaults` updated;
  reactor compile gate catches any missed call sites.

### B1.3 — `ProviderPipeline` SPI adoption

- `ProviderPipeline<RequestT, ResponseT>` SPI signatures rewritten to match
  real provider internals (`RequestBuilder.build(List<Msg>, ModelConfig, boolean)`,
  `ResponseParser.parse(RawT)`, `StreamSubscriber.onChunk(RawT)`,
  `ErrorClassifier.isRetryable(Throwable)`).
- `AnthropicProvider` now `implements ProviderPipeline<String, String>` with
  all 4 stages exposed through `requestBuilder()` / `responseParser()` /
  `streamSubscriber()` (stateless prototype — real subscriber is sink-bound
  and constructed per-stream) / `errorClassifier()`.
- `OpenAIProvider` same treatment (`RawStreamingModelProvider` +
  `ProviderPipeline<String, String>`).
- All 8 delegate classes (`AnthropicRequestBuilder`, `AnthropicResponseParser`,
  `AnthropicErrorClassifier`, `AnthropicSseSubscriber`, and OpenAI equivalents)
  implement the corresponding nested functional interface.
- Evidence: `ProviderPipelineContractTest` (name-based) stays green; provider
  unit + integration tests unchanged and still passing.

### B1.4 — `kairo-skill` physical module split

- New reactor module `kairo-skill` (artifactId `io.github.captaingreenskin:kairo-skill`)
  depending on `kairo-api` + `reactor-core` + `jackson-databind` +
  `jackson-dataformat-yaml` + `jackson-datatype-jsr310` + `slf4j-api`.
- Moved 7 production classes:
  - From `kairo-core/src/main/java/io/kairo/core/skill/`:
    `DefaultSkillRegistry`, `DefaultTriggerGuard`, `SkillChangeHistory`,
    `SkillLoader`, `SkillMarkdownParser`.
  - From `kairo-evolution/src/main/java/io/kairo/evolution/`:
    `InMemoryEvolvedSkillStore`, `SkillContentInjector`.
- Package rename `io.kairo.core.skill.*` / `io.kairo.evolution.{InMemoryEvolvedSkillStore,
  SkillContentInjector}` → `io.kairo.skill.*`.
- Moved 10 test files + classpath resources (`test-skill.md`, `skills/classpath-skill.md`).
- Updated consumers: `kairo-tools`, `kairo-spring-boot-starter-core`,
  `kairo-spring-boot-starter-evolution`, `kairo-examples` pom + source imports.
- Added `kairo-skill` to `kairo-bom`.
- Reactor now has **23 modules** (was 22).
- Circular-dependency sweep via `mvn dependency:tree` — no cycles.
