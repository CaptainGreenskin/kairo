# v0.8 Kickoff: Execution Model + DurableExecution

## 输入前提（来自 v0.7.1）

- 工具结果预算元数据契约已稳定。
- 错误分类字段可在运行时稳定产出。
- 灰度复盘结论允许进入实现阶段。

## Action Backlog

1. 定义执行事件模型（iteration/tool/guardrail/compaction/checkpoint）。
2. 定义 DurableExecution 最小状态快照结构。
3. 定义恢复语义（从最近 checkpoint 恢复、dangling tool call 处理策略）。
4. 定义 replay 输入输出协议（过滤、分页、按 toolUseId 追踪）。
5. 打通事件到 observability 的导出路径（tracer attribute schema）。
6. 增加恢复一致性测试（中断、重启、重放对齐）。

## 建议模块拆分

- `kairo-api`
  - 扩展执行事件与恢复契约接口。
- `kairo-core`
  - 实现执行事件产出与 durable 状态管理。
- `kairo-observability`
  - 对接指标和 tracing 导出。
- `kairo-tools`
  - 补充 replay/inspect 工具（可选）。

## 验收定义

- 恢复后能继续执行且无 tool call 重复提交。
- replay 能准确呈现预算裁剪、错误分类、关键状态转移。
- 关键路径存在自动化测试并可复现。
