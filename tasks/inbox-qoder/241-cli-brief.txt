在当前 workspace（已是 git worktree feature/qodercli-241-http-tool, 基于 main, 只有这一条任务）完成：

新建 kairo-tools/src/main/java/io/kairo/tools/info/HttpTool.java，实现 Java 11+ HttpClient 的 HTTP 客户端工具。

要求：
- 类签名: @Tool(name="http_request", category=ToolCategory.INFORMATION, sideEffect=ToolSideEffect.WRITE) public class HttpTool implements ToolHandler
- @ToolParam 参数：
  - url (required): 请求 URL
  - method (optional, 默认 GET): GET/POST/PUT/DELETE/PATCH/HEAD
  - headers (optional, JSON 对象): 额外请求头
  - body (optional): 请求体字符串
  - timeoutSeconds (optional, 默认 30)
  - followRedirects (optional, 默认 true)
- 用 java.net.http.HttpClient (JDK 标准库)，禁止引入 OkHttp / Apache HttpClient 等新依赖
- 响应体最大 100KB，超过截断且 metadata 标 truncated=true
- 非 2xx 状态码返回 ToolResult.error，但 body 仍要带回，isError=true
- ToolResult metadata 必须含: statusCode, method, url, headers (响应头), truncated, readOnly=false
- SSRF 防护：拦截 localhost / 127.* / 192.168.* / 10.* / 172.16-31.* / ::1
- **必看参考实现**：kairo-tools/src/main/java/io/kairo/tools/info/WebFetchTool.java（同包同风格，照着改；SSRF 黑名单常量提取为共用常量或工具类，不要重复抄）

测试 kairo-tools/src/test/java/io/kairo/tools/info/HttpToolTest.java 必须覆盖：
- GET 200 正常路径
- POST 带 body
- 自定义 headers 透传
- 超时（短 timeout + 慢 endpoint）
- 非 2xx 返回 isError=true 但带 body
- 100KB 截断
- SSRF 拦截（http://127.0.0.1, http://localhost）

不能真实发外网请求。用 JDK 自带 com.sun.net.httpserver.HttpServer 起本地 server（看 WebFetchToolTest.java 用的什么，沿用相同方案）。
